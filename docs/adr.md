## ADR-001: Efficiency Ratio 기반 레짐 감지

**날짜:** 2026-03-06
**상태:** 적용됨
**배경:** 기존 ATR 절대값 기반(ATR < 0.3%) 레짐 감지는 크립토 시장에서 RANGING을 한 번도 감지하지 못함 (3/6 하루 0회). BTC ATR이 항상 0.8%+ 이상이라 임계값에 도달 불가.

**결정:** Efficiency Ratio(ER) 기반 상대적 측정으로 교체.
- ER = |방향 이동| / 총 이동량 (10-period)
- ER < 0.18 → RANGING, ER > 0.35 → TRENDING
- 중간 영역: BB 폭 < 1.5% → RANGING, 이외 TRENDING

**근거:**
- ER은 0~1 범위의 상대지표 → 자산/시간대 무관하게 작동
- 3/7 적용 후 RANGING 116회 감지 (56% 사이클), 정상 작동 확인
- 3/8 초기 ER_RANGING=0.25가 너무 엄격 → 0.18로 하향 (진입 기회 부족 해결)

**대안 검토:**
- ATR 상대값 (ATR/price): 크립토에서 범위가 좁아 구분력 부족
- ADX: 계산 복잡, 지연 큼, ER 대비 장점 없음

## ADR-002: 수수료 포함 PnL 계산

**날짜:** 2026-03-07
**상태:** 적용됨
**배경:** 일주일간 112건 거래, 겉보기 PnL +$0.10이었으나 실제 수수료 ~$2.50 발생. 수수료를 계산하지 않아 봇이 수익을 내고 있다고 착각함.

**결정:**
1. `calculate_pnl()`에 왕복 수수료(0.10%) 자동 차감
2. `calculate_pnl_raw()` 별도 유지 (SL/TP 가격 비교용)
3. 진입 전 FeeFilter: 예상수수료 > 예상수익 50% 시 차단
4. 청산 로그에 fee, 누적fee 표시

**근거:**
- Backpack taker fee: 0.05% per order, 왕복 0.10%
- 건당 수수료($0.017) > 건당 PnL($0.001) → 구조적 적자
- 수수료 미반영 시 전략 판단 자체가 왜곡됨

**영향:**
- 모든 PnL 수치가 더 보수적으로 변경
- FeeFilter가 수수료 대비 이익 없는 진입 차단

## ADR-003: Flip 거래 제거

**날짜:** 2026-03-08
**상태:** 적용됨
**배경:** 일주일간 flip(방향 전환) 34건, 승률 23%, PnL -$0.86 (전체 손실의 40%).

**결정:** 반대 신호 발생 시 기존 포지션 청산만 수행, 반대 방향 신규 진입 안 함. 청산 후 쿨다운(900초) 강제 적용.

**근거:**
- flip 거래는 기존 포지션을 손실 청산 + 반대 진입으로 이중 수수료 발생
- 반대 신호가 맞아도 즉시 진입하면 타이밍이 나쁨 (추격 매매)
- 쿨다운 후 다시 신호가 나오면 그때 정상 진입이 더 안전

**대안 검토:**
- FLIP_THRESHOLD 추가 상향 (3.5→5.0): 근본 해결 안 됨, flip 자체가 구조적 손실
- 부분 청산 후 flip: 복잡도 증가 대비 효과 불확실

## ADR-004: 포지션 사이즈 상향

**날짜:** 2026-03-08
**상태:** 적용됨
**배경:** $47 자본에서 0.0002 BTC($13.6) 거래 시 TP 수익 $0.03~0.06, 수수료 $0.014. 수수료 비중 40~50%.

**결정:**
1. STRATEGY_TRADE_SIZE: 0.0005 → 0.001 BTC
2. MIN_ORDER_SIZE (BTC): 0.0002 → 0.0005
3. 포지션 캡: max_coins → max_coins × 2.0 (signal_boost 반영)

**근거:**
- 수수료 비중은 포지션 사이즈에 무관 (둘 다 비례)
- 하지만 절대 수익이 커져야 의미 있는 거래 가능
- 최대 포지션 0.002 BTC($136) 시 SL 손실 $0.60 = 자본의 1.3% → 안전

**리스크 분석:**

| 포지션 | Notional | SL 손실 | 자본 대비 | 판정 |
|---|---|---|---|---|
| 0.0005 | $34 | $0.15 | 0.3% | 안전 |
| 0.001 | $68 | $0.30 | 0.6% | 안전 |
| 0.002 | $136 | $0.60 | 1.3% | 허용 |

## ADR-005: TP 하향 조정

**날짜:** 2026-03-08
**상태:** 적용됨
**배경:** 112건 중 TP 도달 2건(1.8%). 대부분 time_limit으로 종료되어 수익 잠재력 구조적 제한.

**결정:**
- Ranging TP: 0.8×ATR → 0.5×ATR
- Trending TP: 1.5×ATR → 1.0×ATR

**근거:**
- ATR ~0.44% ($300). 기존 trending TP = $450 → 40분 내 도달 확률 극히 낮음
- 1.0×ATR($300)이면 30~40분 내 도달 가능성 합리적
- Ranging 0.5×ATR($150)은 빠른 수익 확보 전략
- 수수료(0.10%) 대비 TP 수익: 0.5×0.44% = 0.22% → 수수료의 2.2배, 유효

**대안 검토:**
- 부분 청산 (50% at 0.5×ATR, 나머지 trailing): 구현 복잡, 추후 고려
- TP 없이 trailing stop만 사용: 횡보장에서 수익 확보 불가

## ADR-006: 쿨다운 및 임계값 상향

**날짜:** 2026-03-07
**상태:** 적용됨
**배경:** 하루 40건+ 거래 → 수수료만 $1+ 소모. 거래 빈도 자체가 손실 원인.

**결정:**
- COOLDOWN_SEC: 300s → 900s (15분)
- MIN_HOLD_SEC: 300s → 600s (10분)
- ENTRY_THRESHOLD: 1.0 → 2.0
- FLIP_THRESHOLD: 2.5 → 3.5

**근거:**
- 하루 최대 거래 = 24h ÷ (15min 쿨다운 + 30~40min 보유) ≈ 32건
- 실제로는 필터 체인 때문에 5~10건/일 예상
- 높은 임계값 = 더 확신 있는 신호에서만 진입 → 승률 향상 기대
- 3/7 적용 후: 40건→13건, 승률 42%→54%

## ADR-007: ER 임계값 완화

**날짜:** 2026-03-08
**상태:** 적용됨
**배경:** ER_RANGING=0.25 설정 후 3/8 하루 79%가 RANGING 판정, 진입 2건뿐.

**결정:**
- ER_RANGING: 0.25 → 0.18 (확실한 횡보만 차단)
- ER_TRENDING: 0.40 → 0.35 (추세 판정 완화)
- BB 보조판단: BB < 2.0 → BB < 1.5 (중간 영역 trending 유도)

**근거:**
- ER 분포 분석: 대부분 0.10~0.33에 집중
- ER 0.20~0.25는 약한 방향성 있음 → 차단하면 과도한 필터링
- ER < 0.18이면 진정한 횡보 (방향 이동 < 총 이동의 18%)
- 목표: TRENDING 비율 30~50% (기존 16% → 개선)

## ADR-008: MAX_POSITION_CAP 복원 및 SMA 단축

**날짜:** 2026-03-08
**상태:** 적용됨
**배경:** 3/8 거래 15건 분석 결과:
- Trade #15: LONG 0.002 BTC → SL → -$1.02 (자본의 2.2%). MAX_POSITION_CAP = max_coins × 2.0이 단일 거래 재앙적 손실 유발.
- 13/15 거래가 SHORT. SMA(10) 10시간 lookback이 3/7 하락을 계속 반영해 방향 전환 지연.

**결정:**
1. MAX_POSITION_CAP: max_coins × 2.0 → max_coins (0.001 BTC)
2. SMA period: 10 → 7 (7시간 lookback)
3. TrailBE 미사용 변수(pnl_now) 정리

**근거:**
- 0.001 BTC 시 SL 손실 $0.30 (자본 0.6%) → 안전 범위
- SMA 7은 방향 전환을 ~3시간 빠르게 감지
- SMA 5는 너무 짧아 노이즈 민감, 7이 속도/안정 타협점
- TrailBE는 이미 raw 가격 거리(profit_dist) 사용 중 → 수수료 영향 없음 확인

**리스크 분석:**

| 항목 | 변경 전 | 변경 후 | 영향 |
|---|---|---|---|
| 최대 포지션 | 0.002 BTC | 0.001 BTC | 최대 SL 손실 $1.02→$0.30 |
| SMA lookback | 10h | 7h | 반전 감지 ~3h 빠름 |
| 노이즈 리스크 | 낮음 | 약간 증가 | 7h은 충분히 안정적 |

## ADR-009: 리밋 오더 진입 전환

**날짜:** 2026-03-09
**상태:** 적용됨
**배경:** MFE/MAE 분석 결과, 마켓 오더 수수료(0.10% 왕복)가 거래 수익의 52%를 차지.
TP 0.3×ATR 기준 승률 63.6%에서도 기대값 마이너스 (수수료 때문).
리밋 오더(maker 0.02%)로 전환하면 수수료 비율 21%로 감소, 손익분기 가능.

**결정:**
1. 진입: 마켓 → post-only 리밋 오더 (maker fee 0.02%)
2. 청산: 마켓 유지 (SL/TP/time_limit — 안전 우선)
3. 수수료 계산: TAKER_FEE × 2 → MAKER_FEE + TAKER_FEE = 0.07%

**구현 상세:**
- 진입 가격: LONG=best_bid, SHORT=best_ask (오더북 최우선 호가)
- postOnly=True: 스프레드 교차 시 주문 거부 (taker 수수료 방지 보장)
- Fill 대기: 최대 30초, 2초 간격 폴링
- 미체결 시: cancel 후 스킵 (다음 사이클 재시도)
- 실제 체결가로 SL/TP 설정 (시장가가 아닌 fill_price 사용)

**수수료 영향:**

| 항목 | 변경 전 | 변경 후 | 절감 |
|---|---|---|---|
| 왕복 수수료 | 0.10% | 0.07% | -30% |
| $67 노셔널 기준 | $0.067/거래 | $0.047/거래 | $0.020/거래 |
| 일 15거래 시 | $1.01 | $0.70 | $0.30/일 |

**수정 파일:**
- `backpack_client.py`: get_best_bid_ask, place_post_only_limit_order, get_order_fill_price 추가
- `risk_manager.py`: MAKER_FEE 추가, calculate_pnl/estimate_fee mixed 계산
- `self_improving_bot.py`: 진입부 리밋 오더 + fill 모니터링, 수수료 로깅 혼합
